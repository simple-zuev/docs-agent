Mutation output/result contract alignment audit
Date: 2026-05-13
Scope: audit-only, no runtime behavior change

Context
- Project mode: Path A assisted bounded mode.
- Safe assisted surfaces remain: repo-state, lookup/navigation commands, artifact-state, doc-body-only.
- read-doc, get-file, and read-doc-from-query remain degraded by default for assisted baseline purposes.
- This audit does not promote any degraded read surface.
- This audit does not authorize live Google Drive or Google Docs mutations.

Audit target
Map mutation-related commands to their current result/output style and identify the narrowest output-contract alignment candidate.

Primary finding
The current bounded mutation problem is not primarily write gating. The narrow mismatch is result/output shape.

Several low-level mutation helpers already return structured dictionaries, but command handlers often discard those return values and print ad hoc human-readable success lines.

Several dry-run paths emit a human banner followed by JSON. This is useful for operators but not a clean machine-readable JSON envelope.

Contract gaps

Gap 1: Dry-run output is not clean JSON
- Current style: DRY RUN OK banner followed by JSON payload.
- Effect: operator-readable, but not a single machine-readable JSON document.
- Narrow fix direction: add json_output-aware dry-run emission.

Gap 2: Live success output is mostly human-only
- Current style: command handlers print ad hoc success lines.
- Effect: hard for ChatGPT/Codex/operator wrappers to consume deterministically.
- Narrow fix direction: start with dry-run envelopes only; defer live success envelope standardization.

Gap 3: Low-level structured results are not consistently propagated
- Current style: helpers can return dictionaries, but command handlers may discard them.
- Effect: structured mutation facts are lost at command boundary.
- Narrow fix direction: preserve return dictionaries in JSON mode after dry-run alignment proves safe.

Non-goals for the next production PR
- Do not promote read-doc, get-file, or read-doc-from-query.
- Do not change live Google Drive mutation behavior first.
- Do not route live mutation commands through agent_cli.py.
- Do not perform broad refactors of docs_agent.py or agent_cli.py.
- Do not change write gates, confirmation gates, or safety mode semantics.

Recommended narrow production candidate

First production PR:
- Add clean --json-output support only to dry-run paths for bounded review-safe mutation commands:
  - create-draft-doc
  - append-review-note
  - write-draft-doc

Preferred implementation contour:
- Keep live mutation behavior unchanged.
- Keep write gates unchanged.
- Keep confirmation gates unchanged.
- Make dry-run output choose between:
  - human banner plus pretty JSON for default human mode;
  - clean single JSON envelope for --json-output mode.
- Standard envelope keys:
  - ok
  - command
  - dry_run
  - write_mode_required
  - target identifiers
  - next_safe_step

Conclusion
The next safe step is an audit-only PR containing this contract map. The narrowest later production change should target clean JSON output for dry-run mode on review-scoped mutation commands only. Live mutation success envelopes should be deferred until the dry-run contract is stable.
