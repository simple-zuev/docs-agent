Backend task trace fields audit
Date: 2026-05-15

Problem
- TASK_MODEL.md defines created_at, updated_at, created_by, authority_binding_id, drive_context_id, and notes as minimum task fields.
- The operator backend task responses had equivalent context in some places, but did not expose those minimum fields explicitly.

Decision
- Add the documented task trace fields to TaskSummary so list and details responses share the same minimum task identity contract.
- Populate seed tasks and newly created mock tasks.
- Update updated_at when a task patch changes persisted task state.
- Preserve existing authority_source and drive_object fields for compatibility.

Safety boundary
- Mock backend API contract only.
- No Google Drive or Google Docs behavior is changed.
- No live Google API calls or mutation handlers are introduced.

Validation
- make test-backend: passed, 7 tests.
- make test-all: passed, 56 CLI tests and 7 backend tests. CLI suite still emits the existing Python 3.9 Google dependency warnings.
- git diff --check: clean.

Rollback
- Revert operator_backend task model/store/test changes and this audit note.
