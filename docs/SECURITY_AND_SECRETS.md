# Security and Secrets

## Что считается секретом

- OAuth client secrets
- access tokens
- refresh tokens
- credentials JSON
- service account keys
- локальные конфигурации с чувствительными параметрами

## Что нельзя коммитить

Нельзя коммитить:
- config/client_secret.json
- config/token.json
- любые токены
- любые ключи
- .env с чувствительными данными

## Если секрет был показан

Если секрет:
- был вставлен в чат
- попал в терминальный лог
- был случайно опубликован

его нужно считать потенциально скомпрометированным.

Тогда рекомендуется:
1. перевыпустить секрет
2. удалить старый token
3. пройти авторизацию заново
4. убедиться, что он не попадает в git

## Правила локальной работы

- config/ хранится только локально
- config/ должен быть в .gitignore
- при ревью untracked files отдельно проверять config/

## Практические замечания

1. Самый опасный класс ошибки — не quota, а случайная публикация токенов.
2. Если секрет был показан в чате или логах, его лучше перевыпустить.
